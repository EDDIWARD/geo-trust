# 安卓端与后端云服务器部署指南

## 1. 这份指南解决什么问题

当前你们的联调方式是：

- 电脑启动本地 FastAPI 后端
- 手机连接电脑热点
- 安卓端通过局域网 IP 访问电脑

这种方式适合开发和真机调试，但如果要让：

- 手机不再依赖电脑热点
- 多台手机都能直接访问
- 后续答辩时更稳定
- 后续再接消费者端和管理端

就应该把后端部署到云服务器。

本指南按当前项目结构，给出一套最直接的迁移方法。

## 2. 当前项目里和部署有关的关键点

### 2.1 后端入口

当前后端启动入口在：

- [main.py](D:/Users/Administrator/Desktop/计算机设计/backend/server/main.py)
- [app/main.py](D:/Users/Administrator/Desktop/计算机设计/backend/server/app/main.py)

实际运行命令是在 `backend/server` 目录下执行：

```bash
uvicorn main:app --host 0.0.0.0 --port 8000
```

### 2.2 后端配置

后端配置在：

- [config.py](D:/Users/Administrator/Desktop/计算机设计/backend/server/app/config.py)

当前会读取这些关键环境变量：

- `BASE_URL`
- `DATABASE_PATH`
- `SCHEMA_PATH`
- `SAMPLE_REGIONS_PATH`
- `REGISTER_ENABLED`
- `LOCATION_REQUIRED`
- `REJECT_MOCK_LOCATION`
- `REJECT_EMULATOR`
- `REJECT_DEBUGGER`
- `SIGNING_SECRET`

其中最关键的是：

- `BASE_URL`

因为后端返回给安卓端的：

- `trace_url`
- `qr_code_url`

都是基于 `BASE_URL` 拼出来的。

如果部署到云服务器后还保留默认的 `http://127.0.0.1:8000`，手机虽然可能能调接口，但二维码图片地址和追溯链接一定会错。

### 2.3 安卓端访问地址

安卓端当前基础地址在：

- [gradle.properties](D:/Users/Administrator/Desktop/计算机设计/backend/android-app/gradle.properties)

字段是：

```properties
apiBaseUrl=http://你的地址:8000/
```

构建时会写进：

- [app/build.gradle.kts](D:/Users/Administrator/Desktop/计算机设计/backend/android-app/app/build.gradle.kts)

也就是：

- `BuildConfig.BASE_URL`

## 3. 云服务器部署后的目标结构

建议部署后的结构是：

1. 云服务器运行 FastAPI 后端
2. 云服务器对外提供固定公网 IP 或域名
3. 安卓端把 `apiBaseUrl` 改成公网地址
4. 手机直接通过公网访问后端

例如：

- 后端地址：`http://123.123.123.123:8000/`

或者更推荐：

- 后端地址：`https://api.example.com/`

## 4. 推荐的比赛版部署方案

如果目标是：

- 快速可用
- 容易排错
- 适合比赛演示

建议先用这套方案：

1. 购买一台 Linux 云服务器
2. 系统用 Ubuntu 22.04 或类似版本
3. 先直接部署 `FastAPI + Uvicorn`
4. 第一版可以先用公网 IP + 8000 端口
5. 稳定后再加 Nginx 和域名

如果只求快，先不用一开始就上太多复杂组件。

## 5. 云服务器准备

以 Ubuntu 为例。

### 5.1 需要准备的东西

- 一台有公网 IP 的云服务器
- 能通过 SSH 登录
- 已安装 Python 3.10+

### 5.2 开放端口

至少需要开放：

- `22`，用于 SSH
- `8000`，如果你直接暴露 Uvicorn

如果后面加 Nginx，则开放：

- `80`
- `443`

同时要检查两层防火墙：

1. 云厂商安全组
2. 服务器系统防火墙

## 7. 后端启动步骤

### 7.1 进入后端目录

```bash
cd /opt/geotrust/backend/server
```

### 7.2 创建虚拟环境

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 7.3 安装依赖

```bash
pip install -r requirements.txt
```

依赖文件在：

- [requirements.txt](D:/Users/Administrator/Desktop/计算机设计/backend/server/requirements.txt)

### 7.4 设置环境变量

先给一版最小可用的示例：

```bash
export BASE_URL=http://你的公网IP:8000
export SIGNING_SECRET=请换成你自己的随机长字符串
export REGISTER_ENABLED=true
export LOCATION_REQUIRED=true
export REJECT_MOCK_LOCATION=true
export REJECT_EMULATOR=true
export REJECT_DEBUGGER=true
```

如果你已经有域名，则写成：

```bash
export BASE_URL=https://api.example.com
```

### 7.5 启动服务

```bash
uvicorn main:app --host 0.0.0.0 --port 8000
```

### 7.6 验证接口

服务器启动后，在本机浏览器或手机浏览器访问：

```text
http://你的公网IP:8000/health
```

返回：

```json
{"status":"ok"}
```

说明服务正常。

## 9. 安卓端怎么改成云服务器地址

部署后，你只需要改安卓端的：

- [gradle.properties](D:/Users/Administrator/Desktop/计算机设计/backend/android-app/gradle.properties)

把：

```properties
apiBaseUrl=http://192.168.xx.xx:8000/
```

改成：

```properties
apiBaseUrl=http://你的公网IP:8000/
```

如果已经有域名和 HTTPS，就改成：

```properties
apiBaseUrl=https://api.example.com/
```

然后：

1. Sync Project
2. 重新安装到手机

注意：

- `gradle.properties` 改完后，旧安装包不会自动更新
- 需要重新构建并安装

## 10. 当前 HTTP 联调和 HTTPS 正式部署的区别

### 10.1 当前项目为什么能直接用 HTTP

因为安卓端已经在：

- [AndroidManifest.xml](D:/Users/Administrator/Desktop/计算机设计/backend/android-app/app/src/main/AndroidManifest.xml)
- [network_security_config.xml](D:/Users/Administrator/Desktop/计算机设计/backend/android-app/app/src/main/res/xml/network_security_config.xml)

里放开了明文流量，方便本地联调。

### 10.2 如果上云，推荐尽量用 HTTPS

原因：

- 更像正式系统
- 扫码链接和图片地址更规范
- 减少网络策略问题

如果用 HTTPS，则：

1. 安卓端继续可用
2. 甚至后续可以把明文流量配置逐步收紧

## 11. 推荐的正式一点的部署方式

如果你们希望部署更稳，推荐这套结构：

1. `Nginx` 监听 `80/443`
2. `Nginx` 反向代理到 `127.0.0.1:8000`
3. `Uvicorn` 只监听本机
4. 域名解析到服务器
5. HTTPS 证书挂在 Nginx

这样最终访问地址会更整洁：

- `https://api.example.com/health`
- `https://api.example.com/api/mobile/bootstrap`

同时二维码里生成的：

- `trace_url`
- `qr_code_url`

也会更适合展示。

## 12. systemd 持久化运行

如果不想每次手动开服务，建议用 `systemd`。

示例：

```ini
[Unit]
Description=GeoTrust FastAPI Backend
After=network.target

[Service]
User=root
WorkingDirectory=/opt/geotrust/backend/server
Environment=BASE_URL=https://api.example.com
Environment=SIGNING_SECRET=replace-with-your-secret
Environment=REGISTER_ENABLED=true
Environment=LOCATION_REQUIRED=true
Environment=REJECT_MOCK_LOCATION=true
Environment=REJECT_EMULATOR=true
Environment=REJECT_DEBUGGER=true
ExecStart=/opt/geotrust/backend/server/.venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target
```

假设文件名为：

```bash
/etc/systemd/system/geotrust.service
```

然后执行：

```bash
sudo systemctl daemon-reload
sudo systemctl enable geotrust
sudo systemctl start geotrust
sudo systemctl status geotrust
```

## 13. 常见问题

### 13.1 手机能调接口，但二维码图片加载不出来

优先检查：

1. `BASE_URL` 是否还写成了 `127.0.0.1`
2. 二维码返回的 `qr_code_url` 是否是公网地址
3. 云服务器安全组是否放通
4. `static/qrcodes` 是否已正常生成图片

### 13.2 手机访问超时

优先检查：

1. 云服务器公网 IP 是否正确
2. `8000` 端口是否开放
3. Uvicorn 是否真正监听 `0.0.0.0`

### 13.3 接口能访问，但注册失败

优先检查：

1. `SIGNING_SECRET` 是否配置
2. 数据库路径是否可写
3. 产区数据是否已经导入
4. 云服务器日志里是否有 Python 报错

### 13.4 扫码图片打开了，但扫码内容页没有完整功能

这不是部署问题。

因为当前你们这部分后端主要完成的是：

- 安卓登记接口
- 二维码生成

消费者扫码页属于后续消费者端同学负责的范围。

## 14. 比赛阶段的现实建议

如果你们只是为了：

- 手机不依赖热点
- 多设备稳定联调
- 答辩时减少网络不稳定因素

建议按这个顺序做：

1. 先把当前后端原样部署到云服务器
2. 先用公网 IP 跑通安卓登记
3. 再决定要不要补域名和 HTTPS
4. 后续再接消费者端和管理端

不要一开始就把重点放在复杂运维上。

先保证：

- `bootstrap` 正常
- `validate-location` 正常
- `register-product` 正常
- 二维码图片能正常打开

这四件事稳定，比什么都重要。

## 16. 当前结论

从当前项目状态看，把这部分从“电脑热点联调”迁到“云服务器联调”并不难。

核心不是改代码，而是改部署参数：

- 后端正确对外提供公网地址
- `BASE_URL` 配对
- 安卓端 `apiBaseUrl` 改成公网地址

只要这三点一致，当前安卓端与后端闭环就可以直接迁过去。
